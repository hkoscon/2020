How I do troubleshooting on container, more than docker?

event 2020-06-12 (Day 1) ~ 16:00 - 16:30
explore Main Hall
  • comment Mandarin (English interpretation)
  • network_check Advanced

The holy grail of Cloud Native tech is to have zero vendor lock-in. Container makes everything easier. But even with the easiest platform, sometimes you still need to run into problems. In this talk, Phil will share the knowledge about how to diagnose and troubleshoot container (podman, docker) and container platform (Kubernetes, OpenShift).

Phil Huang

Cloud Native Taiwan User Group / Taiwan

Phil Huang (黃秉鈞) is an active member in Cloud Native Taiwan User Group.

He currently serves as Red Hat Hong Kong / Taiwan Solution Architect. Phil is passionate about implementation, sharing and problem solving. In his free time, he also write a blog, Phil Workspace (blog.pichuang.com.tw/), that share knowledges and write-up posts to the world.
